## Important Note:

If You are going to be starting a new module and therefore decide to make a new Springboot Project using the initializer, change the name of the parent folder of 'src' using the file explorer only (if at all you decide to do this) and then, make sure you move the project file to the desired location and finally, make sure you right click your 'pom.xml' file and click 'add as maven project', otherwise you can't get the application to run. 

### Once all this is done:
Now comes the important part, the only way you will be able to commit to the repo is if you pull the entire exisiting one locally on to your system and then commit from there. In other words, once your brand new project is set up, pull this whole repo into the same directory as the existing brand new project and then commit once (make sure you commit to main) in order to get the new project onto your repo on github. From here you may start working on and commiting to github using this new project as you usually would.



# Coding Shuttle Spring Boot Course Repo

This repository contains my practice code and assignments from a Spring Boot course. It includes hands-on implementations of core Spring Boot concepts such as REST APIs, dependency injection, JPA, and database integration.

## Tech Stack
- Java
- Spring Boot
- Spring MVC
- Spring Data JPA
- MySQL
- Maven

## Contents
- Spring Boot basics
- REST API development
- CRUD operations
- Database integration using JPA/Hibernate
- Practice exercises and assignments
